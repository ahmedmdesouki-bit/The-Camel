## 5. 📈 Base-Rate Engine (replaces "prediction")

For any signal — geopolitical or company-specific — output a structured base rate, never a forecast.

**Template:**
- **Signal:** what triggered this
- **Comparables:** N prior cases (state the sample — small N = low trust)
- **Horizon:** the window measured (e.g., 3–6 months)
- **Hit rate:** % of cases that moved in the expected direction
- **Magnitude:** median and range of move
- **Already priced in:** the honest discount on the obvious read
- **Counter-signals:** what would flip this
- **Overfitting check:** real mechanism, or a curve fit on survivors?
- **→ Output:** a *probability tilt and a position size*, not a buy/sell call

## 9. 📏 Position Sizing & Cash Rule

**Sizing.** Size to the invalidation distance and conviction, not fixed buckets. Bigger size only where the invalidation point is close (risk tightly defined) **and** the base rate is strong with a real sample. On a whole-share platform, "size" is literally *how many shares* — so on a tiny book, one share is already a real allocation decision.

**Tiered cash rule (reconciles v10's 10% drag vs v2's 30% buffer).** The two rules served different scales; v11 makes the threshold scale with the book:

| Book size | Idle-cash target | Rationale |

| **< $1,000 (now)** | Deploy each month's deposit within one DCA cycle; flag if idle cash > **1 monthly deposit (~$100)** | At micro-scale, idle cash is pure drag — there's no meaningful dip to "reserve" for |
| **$1,000 – $10,000** | Hold ~**10–15%** dry powder | Enough to add on real dips without starving compounding |
